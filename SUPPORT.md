## Questions

Before opening an issue:

1. Search existing issues.
2. Check that you are on the latest `main`.
3. Include the command or URL that shows the problem.

Use GitHub Issues for bugs and concrete feature requests.

## Bugs

Include:

- Browser and OS, if the issue is user-facing
- Node and pnpm versions, if the issue is local development
- Steps to reproduce
- Expected vs actual behavior
- Relevant logs with secrets removed

## Security Issues

Do not open public issues for security vulnerabilities.

[Report privately via GitHub](https://github.com/windsornguyen/schedule.pizza/security/advisories/new)
or email [security@schedule.pizza](mailto:security@schedule.pizza). See [SECURITY.md](SECURITY.md).
